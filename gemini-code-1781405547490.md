# 小米智能摄像机3 3K版 RTSP 流转化技术指南

本技术指南汇总了关于**小米智能摄像机3 3K版（云台版）**的核心参数、官方存储局域网限制，以及如何利用开源工具 **go2rtc** 绕过封闭生态，在局域网内将私有协议转化为标准 RTSP 流的完整方案。

---

## 一、 设备核心参数与官方特性

小米智能摄像机3 3K版（云台版）是一款主打高分辨率的家用看护设备，其核心硬件与软件指标如下：

| 核心维度 | 技术参数与功能特性 |
| :--- | :--- |
| **分辨率与画质** | 搭载 **500万像素** 传感器，输出 **3K超清画质**（分辨率为 $2960 \times 1666$）。采用 **H.265 (HEVC)** 视频编码。 |
| **夜视与视角** | 支持超微光全彩技术（微光下保持彩色画面）；全暗环境下启用 8 颗无红体感红外补光灯。云台支持 **360°水平旋转** 与 **109°垂直视角**。 |
| **网络与音频** | 支持 **2.4GHz / 5GHz 双频 Wi-Fi** 连接；内置大口径独立扬声器与智能降噪麦克风，支持 8 米远距离双向语音通话。 |
| **边缘 AI 能力** | 支持**本地 AI 人形侦测**（无需经过云端算力），具备人形移动追踪与米家 App 自定义重点区域看护功能。 |
| **官方存储方式** | 1. **MicroSD卡存储**（最高支持 256GB，建议 U1/Class 10 及以上）。<br>2. **NAS 网络存储备份**（依赖本地 MicroSD 卡，定时将历史 MP4 视频文件同步至局域网内 NAS 或路由器共享文件夹）。<br>3. **米家云存储**（付费增值服务）。 |

> ⚠️ **协议局限性：** > 小米/米家全系家用看护摄像机在官方原生固件下**均未开放 RTSP 或 ONVIF 等通用局域网流媒体协议**。无法直接通过标准流地址接入群晖 Surveillance Station、三方 NVR（网络录像机）或 Frigate、Home Assistant 等第三方看护系统。

## 二、 底层协议解析：什么是 MISS 协议？

为了实现闭环生态、云端安全以及商业化考虑，小米引入了 **MISS (Mi Secure Streaming)** 协议。

* **定义：** MISS 是小米自 2020 年后全面推行的一套专用于其智能家居视频设备（摄像机、可视门铃等）的**私有加密流媒体通信协议**，替代了早期不安全的第三方 P2P 协议。
* **核心机制：** 采用**端到端加密 (E2EE)** 与**动态密钥协商**。每次客户端（米家 App 等）尝试查看视频时，必须先向小米云端服务器验证身份，获取具有时效性的加密密钥（Token）以及设备唯一 ID（`did`）。
* **传输链路：** 拿到密钥后，客户端与摄像头在局域网内部通过 `cs2+udp` 或 `cs2+tcp` 协议尝试建立点对点（P2P）直连打洞。如果内网环境极度复杂，才会回退到通过小米云端服务器中继（Relay）。

## 三、 开源解法：基于 go2rtc 的曲线救国方案

开源流媒体中继工具 **go2rtc（自 v1.9.13 版本起）** 已经通过反向工程，**原生内置了小米 MISS 及 cs2 协议**。其逻辑为：“云端拿密钥，本地拉视频流并转化为标准 RTSP”。

### 3.1 部署准备：Docker Compose 架构
由于小米 P2P 协议在建立本地通信时会动态分配 UDP 端口进行打洞，为了避免复杂的端口映射导致握手失败，**必须将 Docker 容器的网络模式设置为 `host`**。

```yaml
version: '3.8'
services:
  go2rtc:
    container_name: go2rtc
    image: alexxit/go2rtc:master-hardware  # 推荐 hardware 镜像，未来若需转码可利用核显算力
    restart: always
    network_mode: host
    privileged: true
    environment:
      - TZ=Asia/Shanghai
    volumes:
      - ./config:/config  # 挂载配置文件路径
